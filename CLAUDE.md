# Email Agent

Local AI-powered email analysis tool. Monorepo with Turbo.

> **Worktree:** This directory (`main/`) is a git worktree. The bare repo is at `../.bare/`. See global CLAUDE.md for git worktree SOPs.

## Commands

```bash
npm install              # Install all dependencies
npm run build            # Build all packages (core must build first)
npm run dev              # Start all dev servers
npm run start            # Start web UI on port 3847
npm run setup            # Interactive setup wizard (gcloud auth + DB init)
npx tsc -p packages/core/tsconfig.json --noEmit   # Type-check core
npx tsc -p packages/web/tsconfig.json --noEmit    # Type-check web
npx tsc -p packages/cli/tsconfig.json --noEmit    # Type-check CLI
```

### CLI

```bash
npx email-agent fetch              # Fetch unread emails ŌåÆ LanceDB (--scope all|unread, --limit N)
npx email-agent run-action <id>    # Run an action (priority, subscription, junk)
npx email-agent list-actions       # List available actions
npx email-agent serve              # Start web UI
npx email-agent cron setup         # Install crontab for periodic fetching (--interval, --scope)
npx email-agent cron status        # Show current crontab entry
npx email-agent cron remove        # Remove crontab entry
npx email-agent config get <key>   # Read config value (dotted path, e.g. gmail.syncActions)
npx email-agent config set <key> <value>  # Set config value
```

## Architecture

```
packages/
  core/   @email-agent/core    ŌĆö Business logic, Gmail API, LanceDB, agents, actions, analysis
  web/    @email-agent/web     ŌĆö Next.js 15 App Router UI (port 3847)
  cli/    @email-agent/cli     ŌĆö Commander.js CLI tool
```

## Key Patterns

- **Agent system**: Strategy pattern executors (Claude SDK/CLI + Codex/Gemini CLI + DirectAPI + OpenRouter) with AgentRouter; supports streaming via `executeStream()`
- **Action system**: Plugin architecture ŌĆö `*.action.ts` files auto-discovered from built-in + user dirs
- **DB**: LanceDB vector database with Apache Arrow schemas

## Key Files

- `packages/core/src/agents/router.ts` ŌĆö Agent selection logic
- `packages/core/src/actions/runner.ts` ŌĆö Action execution pipeline
- `packages/core/src/db/connection.ts` ŌĆö LanceDB init with Arrow schemas
- `packages/core/src/config/defaults.ts` ŌĆö All default config values and prompt templates
- `packages/core/src/gmail/sync.ts` ŌĆö Shared fetchŌåÆembedŌåÆstore pipeline (used by CLI + web)
- `packages/core/src/gmail/operations.ts` ŌĆö Gmail write operations (trash, spam, labels, read/unread)
- `packages/core/src/actions/apply.ts` ŌĆö Maps action results ŌåÆ Gmail operations, applies them
- `packages/web/src/app/api/` ŌĆö All Next.js API routes
- `packages/core/src/actions/built-in/` ŌĆö Built-in actions
- `~/.email-agent/actions/` ŌĆö User-created actions (auto-discovered)
- `~/.email-agent/settings.json` ŌĆö Runtime config (generated by `setup.sh`)
- `setup.sh` ŌĆö Interactive setup wizard (embedding provider, agent mode, GCP project)

## Gotchas

### Build & TypeScript
- After renaming packages, clean-rebuild core: `rm -rf packages/core/dist packages/core/tsconfig.tsbuildinfo && npx tsc -p packages/core/tsconfig.json` ŌĆö stale incremental cache can produce `.d.ts.map` without `.d.ts`
- Core uses `composite: true`; CLI uses `references: [{ path: "../core" }]` ŌĆö core MUST build before CLI type-checks
- Web tsconfig needs `lib: ["ES2022", "DOM", "DOM.Iterable"]` (base tsconfig only has ES2022)

### Imports & Modules
- Web resolves `@email-agent/core/*` subpaths via tsconfig `paths` to source files
- CLI imports from `@email-agent/core` barrel export only (no subpath imports) due to rootDir constraint
- `actions/built-in/index.ts` is a static barrel for webpack ŌĆö new built-in actions must be added here too

### Database
- LanceDB `createEmptyTable()` requires Apache Arrow `Schema`/`Field` objects, NOT plain JS objects
- DB record interfaces need `[key: string]: unknown` index signatures for `table.add()`

### Web (Next.js)
- `next.config.ts` has webpack `extensionAlias` (`.js` ŌåÆ `.ts/.tsx/.js`) ŌĆö required because core uses `.js` extensions but web resolves to `.ts` source via tsconfig `paths`
- Dynamic filesystem patterns (`readdir`, `import.meta.url` dirs, dynamic `import()`) in core don't work in webpack ŌĆö use static imports in web routes (e.g. `ActionRegistry.loadStatic()`)
- `packages/web/package-lock.json` is a **symlink** to `../../package-lock.json` ŌĆö do NOT delete it. Next.js uses it to detect npm as the package manager; without it, it falls back to globally-installed pnpm and fails with `ENOWORKSPACES`
- `packages/web/next.config.ts` has `outputFileTracingRoot` set to monorepo root ŌĆö prevents Next.js from walking up to stray lockfiles in parent directories
- `fetch().json()` needs explicit return type annotations with strict TS + TanStack Query generics

### CLI
- CLI `dev` script uses `tsc --watch` (not `tsx src/index.ts`) because Commander.js CLIs exit immediately without args, which turbo's `persistent: true` treats as failure
- Use `execFileSync`/`execFile` (NOT `execSync`/`exec`) in CLI commands ŌĆö security hook blocks shell injection patterns

### Other
- `node-notifier` types are strict ŌĆö only `title`, `message`, `wait` are valid notification props
- `setup.sh` hardcodes a `settings.json` template ŌĆö new config fields must be added there too, or they won't appear for fresh installs running setup

### Agent Executors (Spawning CLIs)
- Claude CLI: `--system-prompt` (NOT `--system`), `--output-format stream-json` for streaming, NO `--max-tokens` flag (use `--max-budget-usd` instead)
- **Large `--system-prompt` args hang the CLI** ŌĆö combine system prompt into the user prompt instead (see `claude-executor.ts`)
- **Use `spawn` not `execFile`** for Claude CLI ŌĆö `execFile` can get killed (exit 143/SIGTERM) by Node.js for long-running processes
- When spawning `claude` CLI from a process inside Claude Code, strip `CLAUDECODE` env var or it refuses as "nested session" ŌĆö use `cleanEnv()` in `claude-executor.ts`
- Core module changes (via tsconfig `paths`) may not hot-reload in Next.js ŌĆö restart `npm run dev` after modifying `packages/core/` source

## Adding a New Agent or Embedding Provider

1. `config/types.ts` ŌĆö Add to `AgentId` or `EmbeddingProvider` union
2. Create executor (`agents/<name>-executor.ts`) or add branch in `db/embeddings.ts`
3. Register in `agents/router.ts` executors record + `fallbackOrder`
4. Export from `agents/index.ts`
5. Add env vars to `.env.example` + `.env`
6. Add option to `setup.sh` embedding/agent prompts
7. OpenAI-compatible APIs: reuse `openai` SDK with `baseURL` override ŌĆö no new deps

## Adding a New Config Section

1. `config/types.ts` ŌĆö Add interface + field to `AppConfig`
2. `config/defaults.ts` ŌĆö Add default value to `defaultConfig` (existing installs get this via shallow merge in `loadSettings()`)
3. `config/index.ts` ŌĆö Export the new type
4. `setup.sh` ŌĆö Add prompt + include in generated `settings.json` template
5. `packages/web/src/app/settings/page.tsx` ŌĆö Add tab/controls (extract from `local` state with `as` cast)

## Code Style

- ESM throughout (`"type": "module"`, `.js` extensions in imports)
- Strict TypeScript with `noUncheckedIndexedAccess`
- No default exports except action plugin files (`*.action.ts`)

## Environment

See `.env.example` for all variables. Key ones:
- `OPENAI_API_KEY` ŌĆö Required for OpenAI embeddings (text-embedding-3-small)
- `OPENROUTER_API_KEY` ŌĆö For OpenRouter embeddings (Qwen3) + LLM access
- `GCP_PROJECT_ID` ŌĆö Required for Gmail API access
- `AGENT_MODE` ŌĆö "all-agents" | "hybrid" | "direct-api"
- `PREFERRED_AGENT` ŌĆö "claude-sdk" | "claude" | "codex" | "gemini" | "openrouter"
- `ANTHROPIC_API_KEY` ŌĆö Optional, enables `claude-sdk` agent (Agent SDK direct API calls)
